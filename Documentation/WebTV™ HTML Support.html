 This document describes the HTML document elements and attributes supported by the WebTV Browser. The defintion of the HTML elements supported by the WebTV browser are defined by several sources: HTML 2.0 (rfc1866), HTML 3.0 (3/28/95), Netscape HTML extensions to 2.0 and 3.0, InfoExplorer HTML extensions, and extension made by the WebTV browser itself.


Each tag and attribute description, includes a reference to source of the HTML definition. These references are:

HTML 2.0 HTML 2.0 as defined in rfc1866
HTML 3.0 HTML 3.0 as defined by W3C 3/28/95 memo
Netscape Extensions supported by the Netscape 1.1 or 2.0b3 browser
IExplorer Extensions supported by Microsoft's Internet Explorer 2.0
WebTV Extensions unique to the WebTV browser.

Notes:

Several style elements indicate that they render text as italic. Currently, the WebTV browser does not support rendering of italic text. All of the these elements are currently rendered as normal.




Supported HTML Document Elements



HTML (HTML 2.0, 5.1)

Contains the head and body of the document.

HEAD (HTML 2.0, 5.2)

Contains information about the document. None of the elements in HEAD are displayed as document content.

TITLE (HTML 2.0, 5.2.1)

Contains the title of the document. The title is displayed in the title area of the control panel.


BASE (HTML 2.0, 5.2.2)

Specifies a base URL for the document from which all relative paths should be based.

Attributes:

HREF (HTML 2.0, 5.2.2)
The base URL.
META (HTML 2.0, 5.2.5)

Contains the extensible meta information about the document.

Attributes:

HTTP-EQUIV (HTML 2.0)
Binds the element to an HTTP header field. An HTTP server may use this information to process the document. In particular, it may include a header field in the responses to requests for this document: the header name is taken from the HTTP-EQUIV attribute value, and the header value is taken from the value of the CONTENT attribute. HTTP header names are not case sensitive.

CONTENT (HTML 2.0)
The value of the name/value pair.
Example:
<META HTTP-EQUIV="refresh" CONTENT="12; URL=http://foo.bar/blatz.html">
This will automatically load the document specified by URL after 12 seconds.

 BGSOUND (IEXPLORER)
This tag specifies the background sound for this document.

Attributes:

SRC (HTML 2.0)
Specifies the URL of the sound source file

LOOP (IEXPLORER)
Specifies how many times to loop. The format is:
<BGSOUND SRC="xxx" LOOP=4>
or
<BGSOUND SRC="xxx" LOOP=INFINITE>

 BODY (HTML 2.0, 5.3)

Contains the content of the document.

Attributes:

BACKGROUND (HTML 3.0)
 The purpose of this attribute is to specify a URL pointing to an image that is to be used as a background for the document. This background is tiled to provide a background for the entire document.
<BODY BACKGROUND="metal/brushed_aluminum.gif">Document here</BODY>

BGCOLOR (Netscape)
Specifiy the background color for the document.
The format is:
<BODY BGCOLOR="#rrggbb">Document here</BODY>
where "#rrggbb" is a hexadecimal red-green-blue triplet used to specify the background color.

TEXT (Netscape)
This attribute is used to control the color of all the normal text in the document. This basically consists of all text that is not specially colored to indicate a link. The format of TEXT is the same as that of BGCOLOR.
<BODY TEXT="#rrggbb">Document here</BODY>

LINK (Netscape)
This attribute lets you control the coloring of link text. The format of LINK is the same as that of TEXT.
<BODY LINK="#rrggbb">Document here</BODY>

VLINK (Netscape)
 This attribute lets you control the coloring of visited link text. The format of VLINK is the same as that of TEXT.
<BODY VLINK="#rrggbb">Document here</BODY>

 XSPEED, YSPEED (WebTV)
 These attributes let you specify a speed to scroll the background image in the X and Y direction as pixels per second.
<BODY XSPEED=1 YSPEED=1>Document here</BODY>

HSPACE, VSPACE (WebTV)
 These attributes let you specify the fixed margin maintained around the documentcontent on the screen. HSPACE specifies the horizontal, or left and right margins. VSPACE specifies the vertical, or top and bottom margins.
<BODY HSPACE=10 VSPACE=20>Document here</BODY>

 SONG (WebTV)
 This attributes specifies the URL of a MIDI file to be played for the document.

 H1 ... H6 (HTML 2.0, 5.4)

Specifies the heading elements, <H1> through <H6>, which denote section headings.
<BODY>
<H1>Top level heading</H1>
<H2>Second level heading</H2/
</Body>

Attributes:

ALIGN (HTML 3.0)
 Supports alignment of the heading. Possible values are LEFT, RIGHT and CENTER. JUSTIFY is not supported by WebTV. Alignment affects only this heading.

CLEAR (HTML 3.0)
This attribute specifies that the heading should move down to a point "clear" of floating images. CLEAR=left will move vertically down until you have a clear left margin (no floating images). CLEAR=right does the same for the right margin, and CLEAR=all moves down until both margins are clear of images.

P (HTML 2.0, 5.5.1)
Specifies the start of a paragraph. Paragraphs are preceeded by one blank line.

Attributes:

ALIGN (HTML 3.0)
 Supports alignment of the paragraph. Possible values are LEFT, RIGHT and CENTER. JUSTIFY is not supported by WebTV. Alignment stays in effect until next block element.

CLEAR (HTML 3.0)
This attribute specifies that the paragraph should move down to a point "clear" of floating images. CLEAR=left will move vertically down until you have a clear left margin (no floating images). CLEAR=right does the same for the right margin, and CLEAR=all moves down until both margins are clear of images.

PRE (HTML 2.0, 5.5.2)

Specifies the start of a pre formatted block. A mono-spaced font is used, and tabs are expanded to 8 space tab stops. The WIDTH attribute is not supported.

ADDRESS (HTML 2.0, 5.5.3)
This element contains such information as address, signatur and authorship, often at the beginning or end of the body of a document. It is rendered indented.

BLOCKQUOTE (HTML 2.0, 5.5.4)
This element contains text quoted from another source. Is preceeded by a blankline, and indented. Example of use:

I think the play ends
<BLOCKQUOTE>
<P>Soft you now, the fair Ophelia. Nymph, in thy orisons, be all
my sins remembered.
</BLOCKQUOTE>
but I am not sure.

LI (HTML 2.0, 5.6.1)

This element represents a single item in a ordered, or unordered list.

Attributes:

TYPE (Netscape)
 Controls the type of this list item, and subsequent items in the list. Can take the TYPE values for UL or OL, depending on the type of containing list.

UL (HTML 2.0, 5.6.1)

This element represents an unordered list of bulletted items.

Attributes:

TYPE (Netscape)
 Supports specification of the type of bullet used for the list. Values are TYPE=disc, TYPE=circle, and TYPE=square. TYPE=circle is rendered as a triangle by WebTV.

START (Netscape)
 Specifies a starting level for lists that wish to start at values other than 1. START is always specified in the default numbers, and will be converted based on TYPE before display.

Example:
<UL TYPE=disc>
<LI>First list item
<LI>Second list item
<p>second paragraph of second item
<LI>Third list item
</UL>

 OL (HTML 2.0, 5.6.2)

This element represents an order list of number items.

Attributes:

TYPE (Netscape)
 Supports specification of the type of bullet used for the list. Values are TYPE=disc, TYPE=A (capital letters), TYPE=a (small letters), TYPE=I (large roman numerals), TYPE=i (small roman numerals), or TYPE=1 (default numbers).

START (Netscape)
 Specifies a starting level for lists that wish to start at values other than 1. START is always specified in the default numbers, and will be converted based on TYPE before display.

 DIR (HTML 2.0, 5.6.3)

This element represents a list of short items, typically up to 20 characters. Elements may be display in multiple columns. WebTV displays them in a single column with BOLD style.

MENU (HTML 2.0, 5.6.4)

This element represents a list of items displayed in a compact formant. WebTV displays these lists the same of <UL>

DL, DT, DD (HTML 2.0, 5.6.5)

A definition list is a list of terms and corresponding definitions. Definition lists are formatted with the <DT> term flush left, and the <DD> definition indented. Multiple <DT> elements may be paired with a single <DD> element. Example of use:

<DL>
<DT>Term<DD>This is the definition of the first term.
<DT>Term<DD>This is the definition of the second term.
</DL>

CITE (HTML 2.0, 5.7.1.1)

The <CITE> element is used to indicate the title of a book or other citation. It is rendered as italics.

CODE (HTML 2.0, 5.7.1.2)

The <CODE> element indicates an example of code, rendered in a mono-spaced font.

EM (HTML 2.0, 5.7.1.3)

The <EM> element indicates an empasized phrase. It is rendered in italics.
KBD (HTML 2.0, 5.7.1.4)

The <KBD> element indicates text type by a user. It is rendered in a mono-space font.

 SAMP (HTML 2.0, 5.7.1.5)

The <SAMP> element indicates a sequence of literal characters. It is rendered in a mono-space font.

STRONG (HTML 2.0, 5.7.1.6)

The <STRONG> element indicates strong emphasis. It is rendered in bold.

VAR (HTML 2.0, 5.7.1.7)

The <VAR> element indicates a placeholder variable. It is rendered in italics.


B (HTML 2.0, 5.7.2.1)

The <B> element indicates bold text. It is rendered in bold.

I (HTML 2.0, 5.7.2.2)

The <I> element indicates italic text. It is rendered in italics.


TT (HTML 2.0, 5.7.2.3)

The <TT> element indicates teletype text. It is rendered using a mono-spaced font.

U (HTML 2.0, 5.7.2)

The <U> element indicates underline text. While WebTV recognizes this tag, it current renders it as normal text.

SHADOW (WebTV)

The <SHADOW> element indicates shadowed text. While WebTV recognizes this tag, it current renders it as normal text.

A (HTML 2.0, 5.7.3)

The <A> element indicates a hyperlink anchor. It is rendered using blue text. At least one of the NAME and HREF attributes should be present. Anchors with the HREF attributes present are selectable.

Attributes:

HREF (HTML 2.0)
Gives the URL of the head anchor of a hyperlink.

NAME (HTML 2.0)
Names this anchor, and makes it available as the head of a hyperlink.

SELECTED (WebTV)
Indicates that this anchor should be initially selected.

BR (HTML 2.0, 5.8)

The <BR> element specifies a line break between words.

CLEAR (HTML 3.0)
This attribute specifies that the break should move down to a point "clear" of floating images. CLEAR=left will break the line, and move vertically down until you have a clear left margin (no floating images). CLEAR=right does the same for the right margin, and CLEAR=all moves down until both margins are clear of images.

HR (HTML 2.0, 5.9)

The <HR> element is a divider between sections of text. It is normally rendered as a full width beveled line.

Attributes:

SIZE (Netscape)
The SIZE tag specifies how this the line should be.

WIDTH (Netscape)
Specifies an exact width in pixels, or a relative width measured in percent of document width.The default horizontal rule is always as wide as the page.

ALIGN (HTML 3.0)
 Supports alignment of the divider. Possible values are LEFT, RIGHT and CENTER. Alignment affects only this divider.

NOSHADE
Specifies no fancy shading of your horizontal rule.

IMG (HTML 2.0, 5.10)

The <IMG> element refers to an image, via a hyperlink, presented in the flow of the document.


ALIGN (HTML 2.0)
 Specified the alignment of the image with respect to the surrounding text. The following values are supported:

TOP (HTML 2.0) align the top of the image with the tallest item on the line.
MIDDLE (HTML 2.0) align the center of the image witht the basline of the line.
BOTTOM (HTML 2.0) align the bottom of the image with the baseline of the line.
LEFT (HTML 3.0), the graphic will float down and over to the current left margin, and subsequent text will wrap around the right hand side of the graphic.
RIGHT (HTML 3.0), the graphic aligns with the current right margin and, and text wraps around the left hand side of the graphic.
TEXTTOP (Netscape) align the top of the image with the top of the tallest text in the line. ABSMIDDLE (Netscape) align the middle of the current line with the middle of the image. BASELINE (Netscape) aligns the bottom of the image with the baseline of the current line.
ABSBOTTOM (Netscape) align the bottom of the image with the bottom of the current line.

ALT (HTML 2.0)
 Specifies a text alternative to use if the image cannot be retrieved.

 ISMAP (HTML 2.0)
 Indicates the image is an image map. The image must be contained in an <A> element for the attribute to be recognized.

USEMAP
This attribute specifies the name of a client size image map, <MAP> to use for this image.

SRC (HTML 2.0)
 Specifies the URL of the image resource..

HEIGHT (HTML 3.0)
 Specifies the height of the image in pixels.

WIDTH (HTML 3.0)
 Specifies the width of the image in pixels.

BORDER (Netscape)
 Specifies the border with for the image in pixels. Borders are drawn a anti-aliased bevels.

HSPACE (Netscape)
 Specifies the horzontal space to leave on either side of an image.

VSPACE (Netscape)
 Specifies the vertical space to leave above and below the image.

TRANSPARENCY (WebTV)
 Specifies the transparency of the image as a percentage, where 0 is opaque and 100 in complete transparent.

ANI (WebTV)
 Specifies the the URL of an animation cell resource.

FONT (Netscape)

The <FONT SIZE=value> element supports changes to the size of the font.

Attributes:

SIZE (Netscape)
 Specifies the size of font to use for subsequent text.Valid values range from 1-7. The default FONT size is 3. The value given to size can optionally have a '+' or '-' character in front of it to specify that it is relative the the document baseFONT. The default baseFONT is 3, and can be changed with the BASEFONT element.

BASEFONT

This changes the size of the BASEFONT that all relative FONT changes are based on. It defaults to 3, and has a valid range of 1-7.

Attributes:

SIZE (Netscape)
 Specifies the size of font to use as the new base font size.Valid values range from 1-7. The default FONT size is 3.

CENTER (Netscape)
All lines of text between the begin and end of CENTER are centered between the current left and right margins.

FORM (HTML 2.0, 8.1.1)
The <FORM> element contains a sequence of input elements, along with document structuring elements.

Attributes:

ACTION (HTML 2.0)
Specifies the action URI for the form. The action URI of a form defaults to the base URI of the document.

METHOD (HTML 2.0)
Selects a method of accessing the action URI. The set of applicable methods is a function of the scheme of the action URI of the form. Values supported are Query Forms: METHOD=GET and Forms with Side-Effects: METHOD=POST.

TABLE (HTML 3.0)

The <TABLE> specifies a table composed of rows and colums. It can include <CAPTION>, <TH>, <TD>, and <TR> elements.

Attributes:

SRC (HTML 2.0)
 Specifies the URL of the source file

HEIGHT (HTML 3.0)
 Specifies the height of the embedded document in pixels.

WIDTH (HTML 3.0)
 Specifies the width of the embedded document in pixels.

EMBED (Netscape)

The <EMBED> allows other types of documents to be embedded in an HTML document. While WebTV supports the EMBED tag, it currently does not support an specific embedded document types.

Attributes:

SRC
 Specifies the URL of the source file

HEIGHT
 Specifies the height of the embedded document in pixels.

WIDTH
 Specifies the width of the embedded document in pixels.


MAP

Specifies an image map to use in the USEMAP attribute of an IMG. Contains mutliple <AREA> elements.

Attributes:

NAME
Names the map. Used by USEMAP to identify the map.


AREA

Specifies an area within an image map.

Attributes:

SHAPE
Specifies the shape. Currently on RECT is supported.

COORDS
Coordinates of the shape. For RECT coords are specified "top, left, botom, right".

HREF
Specifies the URL corresponding to this area.

NOHREF
Specifies an area for which no action is taken.

Unsupported Netscape Extensions



This section lists the elements and attributes supported by Netscape as extensions to HTML 2.0 and HTML 3.0 that are not supported by WebTV.
BODY (HTML 2.0, 5.2.1)

The following <BODY> attribute is not supported:

 ALINK
This attribute lets you control the coloring of active link text. The format of ALINK is the same as that of TEXT.
<BODY ALINK="#rrggbb">Document here</BODY>
NOBR

The NOBR element stands for NO BReak. This means all the text between the start and end of the NOBR elements cannot have line breaks inserted between them. While NOBR is essential for those odd character sequences you really don't want broken, please be careful; long text strings inside of NOBR elements can look rather odd.

WBR
The WBR element stands for Word BReak. This is for the very rare case when you have a NOBR section and you know exactly where you want it to break. Also, any time you want to give the Netscape Navigator help by telling it where a word is allowed to be broken. The WBR element does not force a line break (BR does that) it simply lets the Netscape Navigator know where a line break is allowed to be inserted if needed.

BIG
The <BIG> element specifies that the enclosed text should be displayed, if practical, using a big font (compared with the current font).

SMALL

The <SMALL> element specifies that the enclosed text should be displayed, if practical, using a small font (compared with normal text).

SUB

The <SUB> element specifies that the enclosed text should be displayed as a subscript, and, if practical, using a smaller font (compared with normal text).

SUP

The <SUP> element specifies that the enclosed text should be displayed as a superscript, and if practical, using a smaller font (compared with normal text)
